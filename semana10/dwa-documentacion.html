Documentación del ejercicio Dynamic Window Approach (DWA)

Durante esta semana he elaborado la documentación de usuario del ejercicio Dynamic Window Approach (DWA), con el objetivo de explicar su funcionamiento, APIs utilizadas y la lógica del algoritmo de navegación local.

La documentación completa oficial del ejercicio puede consultarse en el siguiente enlace

Objetivo del ejercicio

El objetivo es implementar un sistema de navegación local en el que el robot debe alcanzar un objetivo evitando obstáculos de forma segura y suave.

Para ello se generan velocidades candidatas, se simulan trayectorias y se selecciona la mejor opción según una función de coste.

Función de coste

score = α · heading + β · clearance + γ · velocity

Donde:

  • Heading: alineación con el objetivo
  • Clearance: distancia a obstáculos
  • Velocity: velocidad de avance

APIs utilizadas

  • HAL: control del robot (posición, láser, velocidades)
  • WebGUI: visualización y objetivos
  • Frequency: control de frecuencia de ejecución

Control del robot

  • HAL.setV(): establece la velocidad lineal del robot
  • HAL.setW(): establece la velocidad angular del robot

Estas funciones son las encargadas de aplicar el comando de movimiento seleccionado por el algoritmo DWA en cada iteración.

Sensor láser

  • HAL.getLaserData(): obtiene los datos del sensor láser del robot

El láser proporciona 180 medidas de distancia que representan los obstáculos alrededor del robot. Estos datos se utilizan para calcular la variable clearance dentro de la función de coste del DWA.

Concepto clave

A diferencia de otros métodos, DWA trabaja directamente en el espacio de velocidades, evaluando trayectorias futuras en lugar de fuerzas.

Conclusión

Esta documentación permite entender de forma clara cómo funciona el algoritmo DWA, desde la generación de velocidades hasta la selección de la mejor acción en cada ciclo. Es bastante mas extensa, pero solo quería remarcar algunos conceptos clave, el resto se puede apreciar en el enlace del principio.
