Creación de un RADI (RoboticsAcademy Docker Image)

También he aprendido a crear una imagen Docker personalizada (RADI) para Robotics Academy, con el objetivo de modificar dependencias, probar cambios en robots y ajustar entornos de simulación.

Paso 1: Requisitos previos

  • Docker instalado
  • Git instalado
  • Acceso al repositorio de Robotics Academy

Paso 2: Ir al directorio del script

cd /scripts/RADI

Paso 3: Construcción de la imagen RADI

Ejecutamos el script de construcción con las ramas deseadas. Este script permite generar una imagen Docker personalizada del entorno de Robotics Academy. 

./build.sh -a humble-devel -i humble-devel -m humble-devel -r humble -t mi_radi_personal

Significado de cada parámetro:

  • -a (ROBOTICS_ACADEMY): rama del repositorio Robotics Academy que se utilizará. Define la versión del código de ejercicios. Normalmente humble-devel.
  • -i (ROBOTICS_INFRASTRUCTURE): rama del repositorio Robotics Infrastructure. Incluye modelos de robots, mundos y recursos del simulador.
  • -m (RAM - Robotics Application Manager): rama del gestor de aplicaciones de Robotics Academy, encargado de la comunicación entre backend, frontend y simulador.
  • -r (ROS_DISTRO): distribución de ROS usada dentro del contenedor. Actualmente se usa humble.
  • -t (IMAGE_TAG): nombre de la imagen Docker generada. Es el identificador que luego usaremos para lanzar el entorno (por ejemplo: mi_radi_personal).

Cada uno de estos parámetros permite controlar exactamente qué versiones del sistema se están integrando en la imagen final.

Paso 4: Resultado

Se genera una imagen Docker con el tag mi_radi_personal, que será utilizada posteriormente para ejecutar Robotics Academy con configuraciones personalizadas.

Paso 5: Ejecución del RADI

Se puede ejecutar el entorno utilizando dos configuraciones:

  • CPU: dev_humble_cpu.yaml
  • GPU: dev_humble_gpu.yaml

En ambos casos, debemos modificar la imagen por la creada:

image: jderobot/robotics-academy:mi_radi_personal
