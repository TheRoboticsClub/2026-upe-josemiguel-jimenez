Pruebas con datasets KITTI

Durante esta semana se ha trabajado en la fase de validación del algoritmo de Visual Odometry utilizando distintos conjuntos de datos del dataset KITTI. El objetivo principal de esta etapa ha sido evaluar el comportamiento del algoritmo en escenarios variados, con diferentes niveles de dificultad, condiciones de iluminación y tipos de movimiento del vehículo.

El dataset KITTI ofrece dos secciones principales ampliamente utilizadas en tareas de odometría visual: Odometry y Raw Data. Ambas contienen secuencias grabadas desde un vehículo en movimiento, pero con características ligeramente diferentes que permiten probar el algoritmo en contextos complementarios.

Uso de las secuencias del dataset

Dentro de ambas secciones del dataset se dispone de múltiples secuencias que representan recorridos urbanos, carreteras abiertas y situaciones con mayor o menor complejidad visual. Estas secuencias han sido utilizadas para generar diferentes vídeos de prueba, reproduciendo el flujo de imágenes original del vehículo.

El proceso consistió en extraer las imágenes de cada secuencia y generar vídeos reproducibles que simulan la ejecución del sistema en tiempo real, permitiendo así probar la solución de Visual Odometry de forma controlada.

Generación de vídeos de prueba

A partir de las secuencias seleccionadas del dataset KITTI se generaron varios vídeos que posteriormente fueron utilizados como entrada del algoritmo. Cada vídeo representa un escenario diferente dentro del entorno de conducción capturado por el vehículo.

Esto permite evaluar aspectos como la estabilidad del seguimiento, la acumulación de error en trayectorias largas o el comportamiento del algoritmo ante cambios bruscos de iluminación o textura en la escena.

Conclusión

El uso de distintos datasets del KITTI ha permitido ampliar significativamente la fase de pruebas del algoritmo de Visual Odometry. Gracias a la generación de vídeos a partir de las secuencias disponibles en las secciones de Odometry y Raw Data, ha sido posible evaluar el comportamiento del sistema en múltiples escenarios, lo que ayuda a detectar posibles debilidades y validar la robustez de la solución desarrollada.
