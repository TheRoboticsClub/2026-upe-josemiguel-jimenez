Gestión de Pull Request en RA y RI

Durante esta semana se han realizado los Pull Requests correspondientes en los repositorios de Robotics Academy (RA) y Robotics Infrastructure (RI), utilizando la rama de trabajo follow_person_harmonic en RA y follow-person-harmonic en RI.

Los Pull Requests realizados son los siguientes:

  • RA: Pull Request #3651
  • RI: Pull Request #697

1. Actualización previa obligatoria

Antes de crear cualquier Pull Request es imprescindible actualizar la rama principal para evitar conflictos. 

git checkout main
git pull origin main

Después se actualiza la rama de trabajo con la última versión de main: 

git checkout follow_person_harmonic
git rebase main

2. Resolución de conflictos (si existen)

En caso de aparecer conflictos durante el rebase, estos deben resolverse manualmente y continuar el proceso: 

# Resolver conflictos en archivos
git add .
git rebase --continue

3. Pull Request en RA

En RA se crea la rama de trabajo siguiendo la convención del repositorio, utilizando guiones bajos. 

git checkout -b follow_person_harmonic
git add .
git commit -m "Follow person harmonic implementation"
git push origin follow_person_harmonic

Posteriormente se abre el Pull Request hacia la rama principal del repositorio RA.

  • Rama: follow_person_harmonic
  • Repositorio: RoboticsAcademy
  • PR: #3651

4. Pull Request en RI

El flujo en RI es equivalente, asegurando siempre que la rama esté actualizada antes de subir cambios. 

git checkout -b follow-person-harmonic
git add .
git commit -m "Follow person harmonic plugin and world updates"
git push origin follow-person-harmonic
  • Rama: follow-person-harmonic
  • Repositorio: RoboticsInfrastructure
  • PR: #697

5. Importancia del rebase

El uso de git rebase permite mantener un historial limpio y reduce conflictos en la integración entre ambos repositorios. Es especialmente importante cuando se trabaja en paralelo en RA y RI. 

git rebase main

Conclusión

El flujo de trabajo con ramas sincronizadas, actualización previa y resolución de conflictos mediante rebase permite que los Pull Requests en RA y RI se integren correctamente, manteniendo coherencia entre ambos repositorios y evitando problemas en la compilación o integración del sistema.
