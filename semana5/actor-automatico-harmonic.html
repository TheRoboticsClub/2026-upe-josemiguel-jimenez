Elementos integrados en Gazebo Harmonic

En el entorno de Gazebo Harmonic se han incorporado distintos elementos dentro del mundo de simulación para los ejercicios desarrollados en el proyecto. Estos elementos están organizados en dos escenarios independientes, cada uno con un propósito distinto dentro del sistema.

Escenario 1: Persona controlada

En este escenario se utiliza un modelo de persona controlada mediante el plugin desarrollado en la semana anterior (libpersonplugin_harmonic.so), el cual permite habilitar o deshabilitar el movimiento automático a través del parámetro auto_movement. 

<model name="PersonToControl_harmonic">

  <plugin name="person_plugin::Person" filename="libpersonplugin_harmonic.so">
    <auto_movement>false</auto_movement>
  </plugin>

  <static>false</static>

</model>

Este modelo se utiliza para pruebas de interacción directa, donde la persona puede ser controlada externamente dentro del entorno de simulación.

  • Uso del plugin desarrollado en la semana anterior.
  • Control externo del movimiento.
  • Posibilidad de activar o desactivar comportamiento automático.

Escenario 2: Persona automática

En este segundo escenario, la persona se define como un actor dentro del mundo de simulación, con un comportamiento completamente autónomo basado en una trayectoria temporal. 

<actor name="PersonToFollowActor_harmonic">

  <script>
    <loop>true</loop>
    <auto_start>true</auto_start>

    <trajectory type="walk">
      <waypoint>
        <time>15</time>
        <pose>1.0 -7.5 0 0 0 0</pose>
      </waypoint>

      <waypoint>
        <time>37</time>
        <pose>-8.0 -8.0 0 0 0 -1.57</pose>
      </waypoint>

      <waypoint>
        <time>75</time>
        <pose>0.0 0.0 0 0 0 -4.71</pose>
      </waypoint>
    </trajectory>
  </script>

</actor>

Este actor se ejecuta de forma continua en bucle, reproduciendo una trayectoria predefinida mediante tiempos y poses, simulando el movimiento de una persona en el entorno.

  • Movimiento completamente autónomo.
  • Trayectoria basada en waypoints temporales.
  • Ejecución en bucle dentro del escenario.

Incorporación en el world

Ambos elementos, junto con el robot principal del sistema, se incorporan al mundo de simulación mediante etiquetas <include>. 

<include>
  <name>person2follow</name>
  <uri>model://PersonToFollowActor_harmonic</uri>
  <pose>4 9 0 0 0 0</pose>
</include>

<include>
  <name>robot_follow_person</name>
  <uri>model://turtlebot2_harmonic</uri>
  <pose>-1.0 10.0 0.1 0 0 0</pose>
</include>

Esta estructura permite integrar cada elemento de forma independiente dentro del entorno, manteniendo separada la lógica de control, la simulación automática y el robot autónomo.

  • Uso de include para modularidad del mundo.
  • Separación clara entre escenarios.
  • Facilidad para modificar o sustituir elementos individualmente.

Conclusión

La integración de estos elementos permite disponer de un entorno de simulación donde coexisten distintos tipos de agentes: una persona controlada y una persona automática, junto con el robot autónomo del sistema.
