Preparación de rosbags como entrada

Uno de los principales objetivos de esta semana fue comenzar a incorporar una nueva forma de proporcionar datos al ejercicio de Visual Odometry mediante el uso de rosbags. Hasta ese momento, el algoritmo trabajaba principalmente con secuencias de imágenes procedentes del dataset KITTI, pero era necesario adaptar el ejercicio para que pudiera reproducir datos grabados en formato ROS.

Para facilitar este proceso, se desarrolló una herramienta que permite generar rosbags a partir de distintas secuencias de prueba. De esta forma fue posible crear varios archivos que posteriormente servirían como entrada del ejercicio dentro de Unibotics.

Rosbags frente a secuencias de vídeo

La incorporación de rosbags no solo permite reproducir las imágenes de la cámara, sino también incluir información adicional asociada a la secuencia. Una de las más importantes para este ejercicio es la trayectoria real del vehículo (Ground Truth), que puede publicarse junto con las imágenes durante la reproducción del rosbag.

Con el sistema basado únicamente en vídeos era posible procesar la secuencia de imágenes, pero no disponer de información adicional sincronizada con cada fotograma. Esto impedía obtener el Ground Truth directamente durante la ejecución y obligaba a utilizar soluciones alternativas, como la lectura del archivo poses.txt del dataset KITTI, que era una solucion temporal que teniamos antes de incorporar los rosbags.

Gracias al uso de rosbags, toda la información necesaria para el ejercicio queda integrada en una única fuente de datos. Esto permite reproducir tanto las imágenes como la trayectoria real del vehículo de forma sincronizada, acercando el funcionamiento del ejercicio a un entorno real basado en ROS.

Generación de los rosbags

La herramienta desarrollada automatiza la creación de los rosbags, permitiendo disponer de diferentes secuencias listas para ser utilizadas en el ejercicio. Gracias a ello, es posible preparar nuevos escenarios de prueba de forma sencilla sin necesidad de grabarlos manualmente.

El resultado es un conjunto de archivos .bag que contienen la información necesaria para reproducir la secuencia exactamente igual que si procediera de un robot real.

Integración con Unibotics

Una vez generado el rosbag, el siguiente paso consiste en incorporarlo al proyecto. Para ello, únicamente es necesario copiar el directorio completo del rosbag dentro de la carpeta resources del frontend del ejercicio. Cada rosbag está formado por un directorio que contiene el archivo metadata.yaml y uno o varios ficheros .db3, donde se almacenan todos los mensajes grabados.

Posteriormente, durante la ejecución de Unibotics, el rosbag puede reproducirse desde la línea de comandos indicando el nombre del directorio. De esta forma, el ejercicio recibe de manera sincronizada toda la información almacenada en el rosbag, incluyendo las imágenes de la cámara y cualquier otro tópico registrado durante la grabación. 

frontend/
└── resources/
    └── your_rosbag_name/
        ├── metadata.yaml
        └── *.db3

Este mecanismo simplifica la incorporación de nuevos escenarios de prueba, ya que basta con añadir un nuevo directorio de rosbag a la carpeta resources para poder utilizarlo posteriormente durante la ejecución del ejercicio.

Ventajas del nuevo sistema

  • Permite utilizar distintas secuencias de prueba de forma sencilla.
  • Facilita la reutilización de escenarios previamente grabados.
  • Posibilita incorporar información adicional, como el Ground Truth, sincronizada con las imágenes de la cámara.
  • Acerca el ejercicio al funcionamiento habitual de aplicaciones basadas en ROS.
  • Simplifica la incorporación de nuevos conjuntos de datos sin necesidad de modificar el ejercicio.

Conclusión

El trabajo realizado durante esta semana supone el primer paso para incorporar el soporte completo de rosbags dentro del ejercicio de Visual Odometry. La herramienta desarrollada permite generar nuevas secuencias de prueba e integrarlas fácilmente en Unibotics. Además, el uso de rosbags abre la puerta a transmitir información adicional, como el Ground Truth del vehículo, algo que no era posible utilizando únicamente secuencias de vídeo y que resultará fundamental para la versión final del ejercicio.
