Referencias utilizadas para Visual Odometry

Para el desarrollo del nuevo ejercicio de Visual Odometry (VO) hemos tomado como referencia dos ejercicios ya existentes en Unibotics, que nos proporcionan parte de la infraestructura base tanto para la entrada de vídeo como para la visualización en 3D de resultados.

1. Basic Computer Vision - Entrada de vídeo

Del ejercicio Basic Computer Vision hemos reutilizado la forma en la que el usuario puede seleccionar un vídeo local y enviarlo al entorno de ejecución de Unibotics.

Este sistema permite trabajar con imágenes en dos modos:
  • Python API: getImage() y showImage()
  • ROS2: entrada y salida mediante topics

La clase WebGUI actúa como puente entre el navegador y el backend en Python/ROS2, gestionando:

  • Recepción de frames desde el navegador (base64)
  • Decodificación a imágenes OpenCV
  • Publicación en ROS2 si es necesario
  • Envío de imágenes procesadas de vuelta al frontend

Gracias a esta infraestructura, en Visual Odometry podemos utilizar vídeos como entrada inicial, simulando sensores de cámara reales.

2. 3D Reconstruction - Visualización de trayectorias

Del ejercicio de 3D Reconstruction hemos reutilizado el sistema de visualización en 3D, que nos permite representar trayectorias dentro de un entorno tridimensional.

Este visor 3D se utiliza para mostrar:
  • Trayectoria estimada por el usuario
  • Ground Truth (posición real)

El componente principal es WebGUI3D, que renderiza el escenario 3D y permite dibujar en tiempo real las trayectorias calculadas.

Además, el sistema gestiona eventos como reset del entorno y sincronización del estado del ejercicio.

Aplicación en Visual Odometry

En el nuevo ejercicio de Visual Odometry combinamos ambas infraestructuras:

  • Entrada de vídeo basada en Basic Computer Vision
  • Visualización 3D basada en 3D Reconstruction

Esto nos permite construir un sistema donde:

El usuario procesa imágenes de vídeo para estimar movimiento, y posteriormente se representan en 3D tanto la trayectoria estimada como la real (ground truth), facilitando la comparación directa.

Conclusión

La reutilización de estas dos bases ha permitido acelerar el desarrollo del ejercicio de Visual Odometry, aprovechando infraestructura ya validada para entrada de vídeo y visualización 3D.
