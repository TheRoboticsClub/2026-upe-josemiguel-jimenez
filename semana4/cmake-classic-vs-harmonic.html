Problemas de compatibilidad en CMake entre Gazebo Classic y Harmonic

Durante el desarrollo del proyecto se detectó un problema importante de compatibilidad entre el plugin de Gazebo Classic y el nuevo plugin desarrollado para Gazebo Harmonic.

Origen del problema

El objetivo inicial del proyecto era mantener la versión funcional en Gazebo Classic mientras se desarrollaba la migración a Harmonic. Sin embargo, ambos plugins coexistían dentro del mismo sistema de compilación CMake, lo que generaba conflictos de dependencias y compilación.

El problema principal era que:

  • El plugin Classic dependía de gazebo (Gazebo Classic).
  • El plugin Harmonic dependía de gz-sim, gz-plugin y gz-math.
  • Ambos no podían coexistir correctamente en el mismo build sin conflictos.

Fragmento del CMake afectado

Inicialmente, el CMake intentaba compilar ambos plugins: 

#add_library(personplugin amazon_hospital/src/person.cpp)
#ament_target_dependencies(personplugin "rclcpp" "gazebo" "pluginlib")

Mientras que el plugin de Harmonic sí estaba activo: 

add_library(personplugin_harmonic follow_person_harmonic/src/person_harmonic.cpp)

target_link_libraries(personplugin_harmonic
  gz-sim8::gz-sim8
  gz-plugin2::gz-plugin2
  gz-math7::gz-math7
  Threads::Threads)

Solución temporal aplicada

Para poder seguir avanzando con el desarrollo y validar el funcionamiento del sistema, se tomó una decisión temporal:

  • Comentar completamente el plugin de Gazebo Classic en el CMake.
  • Dejar únicamente activo el plugin de Gazebo Harmonic.
  • Permitir así que el proyecto compilara sin conflictos de dependencias.

Esta solución permitió verificar que el resto del sistema funcionaba correctamente en Harmonic, aunque se perdió temporalmente la compatibilidad con Classic.

Resultado

Tras este cambio, el proyecto pudo compilar correctamente y ejecutar la simulación en Gazebo Harmonic, validando el comportamiento del nuevo plugin sin interferencias del código antiguo.

Conclusión

Este problema evidenció la dificultad de mantener compatibilidad entre dos versiones de Gazebo con arquitecturas completamente diferentes. La coexistencia de plugins Classic y Harmonic en el mismo sistema de compilación no es viable sin separación clara de builds.

La solución adoptada permitió continuar el desarrollo, priorizando la migración a Harmonic mientras se mantenía el código Classic como referencia desactivada.
