Semana 1 - 16/02 - 20/02

Durante esta primera semana he seguido la guía para realizar el despliegue D1 de Unibotics. A continuación detallo los pasos y mi progreso:

Paso 1: Preparación del entorno

Lo primero que hice fue crear un entorno virtual, para evitar incompatibilidades con lo previamente instalado en mi máquina. Este se puede activar con el siguiente comando:

source ~/.virtualenvs/academy-venv/bin/activate

Paso 2: Clonación del repositorio e instalación de dependencias

El siguiente paso seria clonar el repositorio en un directorio que vamos a crear:

mkdir -p ~/jderobot/unibotics-webserver
git clone --recurse-submodules git@github.com:jderobot-hub/unibotics-webserver.git ~/jderobot/unibotics-webserver

A continuación debemos instalar los paquetes necesarios:

cd ~/jderobot/unibotics-webserver
pip install -r utils/requirements.txt

Paso 3: Creación de zip de common y configuración de Webpack

Para que el front-end de Unibotics funcione correctamente, es necesario preparar la carpeta common que contiene todas las dependencias de las interfaces. Primero, nos movemos al directorio correspondiente:

cd unibotics/static/RoboticsAcademy/common

Después, se crea un zip de common que incluye las interfaces necesarias (console_interfaces, gui_interfaces y hal_interfaces) para que Webpack pueda construir todas las dependencias del front-end.

cd console_interfaces
zip -r ../common.zip console_interfaces/
cd ..
cd gui_interfaces
zip -r -u ../common.zip gui_interfaces/
cd ..
cd hal_interfaces
zip -r -u ../common.zip hal_interfaces/
cd ../..
mv common/common.zip react_frontend/src/common.zip
cd ../..

Finalmente, se instala y configura NVM (Node Version Manager) para usar la versión correcta de Node.js (17) y se lanza Webpack en modo desarrollo para probar la aplicación.

npm run dev
